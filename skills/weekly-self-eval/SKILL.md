# weekly-self-eval

## Purpose

Catch agent calibration drift weekly. Sharp prompts; concrete answers; output mineable for monthly trend review.

A vacuous self-eval is failure. Each question gets a specific answer or "no, none observed." Skip none.

## Inputs

Read the past 7 days of:

- `~/agents/republic/.watcher/signal_grade_log.jsonl` — graded events, telegram_sent flags, ack data
- `~/brain/signals/auto_review_triggers.md` — additive stale-review refresh candidates from `signal-grade`; if missing, record "no trigger file observed" rather than creating a placeholder
- `~/brain/signals/misgrades.md` — human ack / misgrade patterns from `ack-signal`; if missing, record "no misgrade file observed" rather than creating a placeholder
- `~/brain/` git log: `git -C ~/brain log --since="7 days ago" --oneline`
- `~/brain/meta/pipeline-status-snapshot-*.md` — daily health snapshots (last 7 dates)
- `~/.hermes/sessions/` — recent agent sessions (filter by `mtime` within 7 days; sample for capitulation/pushback patterns)
- `~/brain/meta/weekly-self-eval/` — last week's entry, if any (carry forward unfinished action items)
- `~/brain/meta/decisions.md` — recent deviation entries (sanity check: are documented deviations being honoured?)

## Phase 1 — Answer the 6 questions

Each question gets a specific answer or "no, none observed." No vacuous summaries.

### 1. What did I capture this week that mattered? What did I miss?

List 3-5 items captured well — each with one-line rationale (why it mattered).
List 1-3 items missed — each with what should have been done.

### 2. Did Jack repeat himself? (Sign of memory drift or mis-filing.)

Scan recent sessions for repeat questions or context Jack had to re-supply. If yes — what topic; what should have been remembered; where it should have been filed.

### 3. Did I push back when I should have? Where did I capitulate to weak reasoning?

Specific instances. Look for moments where Jack pushed back on a claim and the agent folded without auditing. If "no examples either way" — say so and move on.

### 4. Was there something we forgot to do?

Any follow-up that was implied or promised and never actioned. Re-surface here. Cross-reference last week's action items and `~/brain/signals/auto_review_triggers.md`: which were closed, which carried forward, and which stale-review candidates need human-visible follow-up.

### 5. What rule failed this week that I should propose?

Skillify candidates: things that went wrong and need a structural fix. Propose: file location + one-line rule. Don't just observe — propose.

### 6. What's still uncertain or unverified that I claimed with confidence?

Any claim made without source / citation / verification that should be flagged. Examples: stale memory references, unanchored estimates, unverified file paths.

## Phase 2 — Output

Write to `~/brain/meta/weekly-self-eval/{YYYY-MM-DD}.md` (atomic write: tempfile + os.replace). Use this structure:

```markdown
# Weekly self-eval — week ending {YYYY-MM-DD}

Generated by `weekly-self-eval` skill, {timestamp BST}.

## What mattered (captured well)
- ...

## Missed
- ...

## Drift signals (Jack repeated himself)
- ...

## Capitulation review
- ...

## Forgot to do
- ...

## Rules to propose
- ...

## Unverified claims
- ...

## Action items (max 5; carry forward to next week's eval if untouched)
- ...
```

If the directory `~/brain/meta/weekly-self-eval/` does not exist, create it.

## Phase 3 — Telegram heads-up

Send one short message via Keith bot: `Weekly self-eval written: {filename}. {N} action items.` No body in the message — Jack reads the file when ready. If `N == 0`, still send the heads-up so Jack knows the eval ran.

## Anti-patterns

- **Vacuous summaries.** "This week was productive" is failure. Each question gets a specific answer or explicit "no, none observed."
- **Self-congratulation.** Don't list things-the-agent-did-well that don't translate to behavior change.
- **Vague action items.** Every action carries a file path, command, or specific next step.
- **Skipping Phase 1 questions.** All 6 always get answered, even if some are "no examples observed."
- **Re-deriving last week's action items.** Read last week's file first; carry forward unfinished items rather than re-discovering them.
